group_by

So far, we have applied vectorized operations to data.frames:

  • select: Keep and drop columns
  • mutate: Create new columns
  • filter: Keep and drop rows

For each of these, we have operated on the whole data set

Often, we instead want to operate on chunks of the data at a time

group_by

Recall our penguins:

penguins
# A tibble: 344 × 3
  species bill_length_mm body_mass_g
  <fct>            <dbl>       <int>
1 Adelie            39.1        3750
2 Adelie            39.5        3800
3 Adelie            40.3        3250
# ℹ 341 more rows

Heaviest penguin:

penguins |> slice_max(body_mass_g)
# A tibble: 1 × 3
  species bill_length_mm body_mass_g
  <fct>            <dbl>       <int>
1 Gentoo            49.2        6300

group_by

How can we get the heaviest penguin of each species?

Conceptually:

  • Split the data into three groups
  • Apply slice_max to each group separately
  • Combine the three rows back together

Sometimes called the split-apply-combine paradigm

group_by

Implemented using the group_by adverb

  • Doesn’t do anything on its own
  • Changes the behavior of other functions
penguins |> group_by(species) |> slice_max(body_mass_g)
# A tibble: 3 × 8
# Groups:   species [3]
  species   island bill_length_mm bill_depth_mm flipper_length_mm body_mass_g

2 Chinstrap Dream            52            20.7               210        4800
3 Gentoo    Biscoe           49.2          15.2               221        6300
# ℹ 2 more variables: sex <fct>, year <int>

Effects of group_by

group_by changes behavior of aggregating functions:

  • min(), max(), n(), sum(), prod(), etc.

Many other operations use these internally

Effects of group_by

Recall

penguins |> slice_max(body_mass_g)

is equivalent to:

penguins |> filter(body_mass_g == max(body_mass_g, na.rm=TRUE))

or even

penguins |> 
    mutate(mass_max = max(body_mass_g, na.rm=TRUE))
    filter(body_mass_g == mass_max)

Effects of group_by

Looking at the first step

penguins |> 
    mutate(mass_max = max(body_mass_g, na.rm=TRUE))
# A tibble: 344 × 3
  species body_mass_g mass_max
  <fct>         <int>    <int>
1 Adelie         3750     6300
2 Adelie         3800     6300
3 Adelie         3250     6300
# ℹ 341 more rows

With group_by, a different max for each group

penguins |> 
    group_by(species) |>
    mutate(mass_max = max(body_mass_g, na.rm=TRUE))
# A tibble: 344 × 3
# Groups:   species [3]
  species body_mass_g mass_max
  <fct>         <int>    <int>
1 Adelie         3750     4775
2 Adelie         3800     4775
3 Adelie         3250     4775
# ℹ 341 more rows

Effects of group_by

Hence,

penguins |> 
    group_by(species) |>
    mutate(mass_max = max(body_mass_g, na.rm=TRUE))
    filter(body_mass_g == mass_max)

rowwise Operations

Rarely, you may need to apply an operation to each row separately without vectorization

rowwise lets you do so - makes each row its own group

# A tibble: 344 × 4
# Rowwise: 
  sex    island    flipper_length_mm body_mass_g
  <fct>  <fct>                 <int>       <int>
1 male   Torgersen               181        3750
2 female Torgersen               186        3800
# ℹ 342 more rows

Note: This is a bit dangerous - vectorization is important for data integrity
